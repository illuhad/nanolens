About

Nanolens is an application to calculate gravitational lensing effects (currently microlensing magnification patterns, but additional features are planned for the future), with a focus on

  • Modern codebase: nanolens is a modern, object-oriented application written in standard-compliant C++11.
  • Generality: Features are implemented in the most general (but still efficient) way possible to allow for easy reuse for different applications.
  • Performance: Fully parallel MPI implementation, Starting with nanolens 3: Optional additional support for GPU acceleration via CUDA, support for GPU clusters.
  • Powerful: nanolens is designed to be an extendible collection of different methods algorithms rather than an implementation of one specific method.
  • User-friendly, flexible configuration: The application is fully controlled via a xml-file. No changes to the code itself are required to alter the solved problem. Results are stored in easy-to-use formats, such as FITS images.

Installation

Dependencies

For a successful build, you must have the following software installed

  • cmake [used as build system]
  • A C++ compiler with C++11 support. gcc >= 4.8 should be fine. [for the new C++ features used in the code]
  • cfitsio [used to read/write fits images]
  • A working MPI implentation, e.g. OpenMPI [required for the parallelization]
  • the boost C++ libraries, especially
    • boost::mpi [C++ interface to MPI]
    • boost::serialization [used to serialize C++ datatypes for MPI data transfers]

Additionally, the following optional dependencies exist:

  • CUDA [for GPU acceleration, available from nanolens 3.0 onwards]
  • Doxygen [if you want to generate code documentation]

Configuration and compilation

Configuration and compilation can be done with the following commands mkdir build; cd build cmake <source directory> make

If you wish to use a more graphical frontend, replace cmake by ccmake in the above commands. ccmake provides a relatively intuitive interface to configure the project, and enable optional features such as CUDA support. By default, CUDA support is disabled.

To generate code documentation, type make doc

Using nanolens

You should now have one program called nanolens and one program called nanopostprocess as well as an example configuration file nanolens.xml

The nanolens xml file

The problem that nanolens solves is defined in a xml file. An example xml file is delivered with the code.

nanolens

The nanolens program executes the configuration of the supplied configuration file. This includes lensing calculations as well as any postprocessing steps (see below).

nanopostprocess

Like nanolens, nanopostprocess reads in a configuration file, but executes only the postprocessing steps. This may be useful if you want to convolve an already calculated magnification pattern with a different source profile without redoing the calculation of the magnification pattern.
